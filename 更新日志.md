# 更新日志

## v1.2.1 (2026-07-13)

### CI/CD 修复

- **适配新 Jenkins 服务器环境**：将 `DOCKER_BUILDKIT=0` 从 environment 块改为 Build 阶段的内联设置
  - 原先通过 environment 块设置 `DOCKER_BUILDKIT = '0'` 在新 Jenkins 服务器（38.92.9.207）上未生效
  - 改为 `DOCKER_BUILDKIT=0 docker build` 内联方式，与 EtaMusicPlayer 项目的构建方式保持一致
  - 构建阶段使用经典 Docker 构建模式（非 BuildKit），确保在新服务器上稳定构建并推送镜像
  - 其他配置保持不变（WEB_PORT=8199、GAME_PORT=34197、数据卷、Factorio 卷、端口映射、环境变量等）

## v1.2.0 (2026-07-09)

### 新增功能

- **保存当前游戏**：在「存档管理」页新增「保存当前游戏」按钮
  - 服务器运行时可用，点击后通过 Factorio `/server-save` 命令将当前游戏状态保存为存档文件
  - 保存指令发送后等待 3 秒让 Factorio 完成写入，然后自动刷新存档列表
  - 服务器未运行时按钮置灰并显示提示文字
  - 保存的存档会出现在存档列表中，可下载/切换/删除

### 变更内容

- 前端 `SavesView.vue` 新增 `serverRunning`、`savingCurrent` 响应式状态
- 新增 `fetchStatus()` 方法获取服务器运行状态，`onMounted` 时调用
- 新增 `saveCurrentGame()` 方法调用 `/api/server/save` 触发保存，等待 3 秒后刷新列表
- 存档管理页操作区新增「保存当前游戏」按钮，根据服务器状态切换样式与可用性
- 后端无需改动，复用已存在的 `/api/server/save` 端点（发送 `/server-save` 控制台命令）

## v1.1.0 (2026-07-09)

### 新增功能

- **自动暂停开关**：在「设置 → 游戏设置」中新增「无人时自动暂停游戏」滑动开关
  - 开启后（默认）：当没有玩家在线时游戏内时间停止推进，可节省服务器资源
  - 关闭后：即使无人在线，游戏内时间也会继续推进
  - 设置保存后下次启动 Factorio 服务器时生效（server-settings.json 会根据该配置重新生成）

### 变更内容

- 后端 `settings` 表新增 `auto_pause INTEGER NOT NULL DEFAULT 1` 字段，并添加数据库迁移脚本
- `SettingsUpdate` Schema 新增 `auto_pause: bool | None` 字段
- 配置 GET/PUT 接口支持 `auto_pause` 字段的读取与更新，自动完成 bool↔int 类型转换
- `factorio_process.py` 中 `server-settings.json` 的 `auto_pause` 项由硬编码 `True` 改为读取数据库配置
- 前端 `SettingsView.vue` `AppConfig` 接口与默认配置新增 `auto_pause` 字段
- 游戏设置区新增滑动开关组件，附带功能说明文字

## v1.0.0 (2026-07-09)

### 新增功能

- **仪表盘**：服务器运行状态总览、运行时长、在线玩家数、当前存档名称、CPU/内存/磁盘资源监控、快捷启停按钮
- **日志查看**：WebSocket 实时日志流推送、历史日志文件浏览、关键词搜索过滤、暂停/自动滚动控制
- **一键指令**：启动/停止/重启服务器、强制保存、踢出玩家、封禁玩家、发送服务器公告
- **存档管理**：存档列表展示（文件名/大小/修改时间/活跃状态）、拖拽上传存档、下载存档、删除存档、切换活跃存档（自动重启）
- **定时备份**：配置备份间隔和最大保留数量、启用/禁用定时备份、手动触发备份、备份列表、下载/恢复/删除备份
- **版本管理**：检测当前安装版本、从 Factorio 官网检查最新稳定版/实验版、在线下载安装新版本（含进度显示）、自动迁移存档和配置
- **系统设置**：Factorio 安装目录、存档/备份/日志目录、服务器端口、游戏密码、最大玩家数、玩家验证、自动保存间隔/份数

### 技术架构

- 后端：Python 3.11 + FastAPI + Uvicorn + SQLite + APScheduler + aiohttp + psutil
- 前端：Vue 3 + TypeScript + Vite + Tailwind CSS + Lucide Icons
- 前后端合并单端口部署（FastAPI 托管 Vue 构建产物）
- WebSocket 实时推送日志和服务器状态
- Factorio 进程通过 asyncio.subprocess 管理，stdin 发送命令，stdout/stderr 捕获日志

### 部署

- Docker 多阶段构建（Node 前端构建 + Python 后端运行）
- Jenkinsfile CI/CD 流水线（构建 → 推送私有仓库 → SSH 部署）
- Docker Named Volume 持久化 Factorio 数据和管理面板数据库
- Windows start.bat 启动脚本（自动检查端口占用、安装依赖、构建前端）
